Lexicographic Curation Workbench Help

Comprehensive guide to LIFT format and application features

Introduction

Welcome to the Lexicographic Curation Workbench (LCW), a professional tool for creating and managing comprehensive dictionaries using the LIFT (Lexicon Interchange FormaT) standard. This application provides full support for LIFT 0.13 with extensive features designed for lexicographers, linguists, and language documentation specialists.

What is LIFT?

LIFT (Lexicon Interchange FormaT) is an XML-based standard developed by SIL International for storing and exchanging lexical data. It enables:

  • Interoperability: Exchange data between different dictionary tools (FieldWorks, FLEx, WeSay)
  • Data Preservation: Long-term archival using an open, documented standard
  • Flexibility: Support for diverse linguistic structures and custom fields
  • Richness: Capture complex lexical relationships, multimedia, and metadata
Why LIFT Matters for Lexicographers

LIFT is the industry standard for dictionary data exchange. Whether you're documenting an endangered language, creating a bilingual dictionary, or building a comprehensive reference work, LIFT ensures your data remains accessible, portable, and compatible with professional tools used worldwide.

Core Features

Multilingual Support Essential

Every text field in LIFT supports multiple writing systems simultaneously. This is crucial for:

  • Headwords: Enter lexical units in multiple scripts (e.g., Arabic + Latin transliteration)
  • Definitions: Provide definitions in multiple languages (e.g., English + Spanish + French)
  • Examples: Show usage in source language with translations
  • Glosses: Brief translations for quick reference
Example: Multilingual Entry 
Headword: cat [en], chat [fr], gato [es], 猫 [zh]
Definition (English): A small domesticated carnivorous mammal
Definition (Spanish): Un pequeño mamífero carnívoro domesticado

Senses & Subsenses Essential

Organize word meanings hierarchically to capture polysemy and semantic relationships:

  • Multiple Senses: Distinguish different meanings of the same word
  • Subsenses: Create unlimited levels of nested meanings
  • Grammatical Info: Assign part of speech and morphological traits to each sense
  • Semantic Domains: Classify senses by topic or field
  • Sense-Level Relations: Link related meanings (synonyms, antonyms)
Example: Polysemous Word "Bank" 
Entry: bank
  Sense 1 (Noun): Financial institution
    Definition: A place where money is kept and financial services are provided
  Sense 2 (Noun): Riverbank
    Subsense 2.1: Edge of a river
    Subsense 2.2: Slope beside a body of water
  Sense 3 (Verb): To deposit money
    Definition: To place money in a financial institution

Examples & Usage Essential

Provide rich contextual information through examples:

  • Source Language Examples: Natural usage in context
  • Translations: Multilingual translations of examples
  • Source Attribution: Cite where examples come from
  • Notes: Add editorial comments about usage
  • Custom Fields: Track status, register, or other metadata

Pronunciation Professional

Comprehensive phonetic and phonological documentation:

  • IPA Transcription: International Phonetic Alphabet support
  • Audio Files: Link to pronunciation recordings
  • TTS Integration: Generate audio using text-to-speech
  • CV Pattern: Document syllable structure (e.g., CVCV)
  • Tone Marking: Capture tonal information for tonal languages
  • Multiple Media: Attach multiple audio files per pronunciation
Example: Full Pronunciation Documentation 
Word: hello [en]
IPA: /həˈloʊ/
CV Pattern: CVCV
Audio: [Play Recording] (native_speaker_001.mp3)
TTS: [Generate Audio]

Etymology Advanced

Track word origins and historical development:

  • Etymology Type: Borrowing, inheritance, compound, etc.
  • Source Language: Specify origin language
  • Etymological Form: The source word form
  • Gloss: Brief meaning of the source word
  • Comment: Detailed notes on linguistic changes

Advanced Features

Variants & Allomorphs Advanced

Document different forms of the same lexeme. Our system largely follows the SIL Fieldworks approach to variants and allomorphs:

  • Spelling Variants: Alternative spellings (e.g., color/colour)
  • Dialectal Forms: Regional variations
  • Phonological Variants: Different pronunciations
  • Allomorphs: One of two or more complementary morphs which manifest a morpheme in its different phonological or morphological environments
  • Morphological Allomorphs: Different forms based on context
  • Grammatical Information: POS and traits for each variant

Understanding Allomorphs vs Variants

When deciding between calling something an allomorph or a variant:

  • Allomorphs if it is phonologically conditioned
  • Variants for morphologically conditioned or lexically conditioned suppletion

Tip: When deciding between calling something an allomorph or a variant: Allomorphs if it is phonologically conditioned, otherwise make it a variant for morphologically conditioned or lexically conditioned suppletion. For more information about allomorphs and variants, see the Need More Help? section below.

Example: Allomorph Usage

In the LIFT format, allomorphs can be represented as direct variant forms with traits: 

<variant>
  <form lang="en"><text>grass roots</text></form>
  <trait name="morph-type" value="stem"/>
</variant>

Lexical Relations Professional

Create a semantic network connecting related entries:

  • Entry-Level Relations: Main form, component, derivation
  • Sense-Level Relations: Synonyms, antonyms, hypernyms
  • Custom Relations: Define your own relationship types
  • Bidirectional Links: Navigate relationships in both directions
Example: Semantic Relations 
Entry: dog
  Relations:
    - Synonym: canine, hound
    - Antonym: cat
    - Hypernym: animal, mammal
    - Hyponym: puppy, beagle, poodle

Reversals Professional

Essential for bilingual dictionaries - create L2→L1 lookup capability:

  • Reversal Entries: Automatically generate reverse dictionary
  • Main Element: Specify the headword in the target language
  • Grammatical Info: Track POS in both languages
  • Multiple Reversals: Support multiple target languages
Example: Bilingual Entry with Reversals 
Entry: book [en]
  Sense 1 (Noun): A written work
    Reversal [fr]: livre
      Main: livre
      Grammatical Info: Noun (masculine)

Editorial workflow and quality control features:

Custom Fields

Extend LIFT to meet your specific project needs with FieldWorks-compatible custom fields:

Standard Custom Fields

  • Exemplar: Prototypical example form for a sense
  • Scientific Name: Latin names for botanical/zoological terms
  • Literal Meaning: Literal translation of compounds/idioms

Custom Field Types

  • MultiUnicode: Multilingual text fields
  • Integer: Numeric values (frequency, difficulty level)
  • GenDate: Flexible dates (approximate, before, after)
  • ReferenceAtomic: Single selection from custom lists
  • ReferenceCollection: Multiple selections from custom lists

Custom Possibility Lists

Create custom classification systems loaded from your LIFT-ranges file:

  • Status Lists: Pending, Confirmed, Published
  • Register Marking: Formal, Informal, Slang
  • Subject Fields: Medicine, Law, Technology
  • Geographic Tags: Regional classifications

Ranges Editor Professional

Manage controlled vocabularies (ranges) for your dictionary. Ranges define the valid values for fields like grammatical categories, semantic domains, and lexical relations.

What are Ranges?

Ranges are LIFT's controlled vocabulary lists that ensure consistency across your dictionary. They define:

  • Grammatical Information: Noun, Verb, Adjective, etc.
  • Semantic Domains: Hierarchical classification systems
  • Lexical Relations: Synonym, Antonym, Part of, etc.
  • Custom Categories: Domain-specific classifications

Managing Ranges

Access the Ranges Editor from Tools → Ranges Editor to:

  • View all ranges: Browse your controlled vocabularies
  • Create new ranges: Add custom classification systems
  • Edit range details: Update labels and descriptions in multiple languages
  • Manage elements: Add, edit, or delete individual values within ranges
  • Check usage: See which entries use specific range values
  • Safe deletion: Migrate values before deleting to maintain data integrity

Hierarchical Ranges

The Ranges Editor supports two hierarchy patterns:

  • Nested Hierarchy: Elements directly nested within parents (e.g., Noun → Common Noun, Proper Noun)
  • Parent-Based Hierarchy: Flat structure with parent attributes (e.g., semantic domains)

Usage Analysis

Before deleting range elements, the editor shows:

  • Usage count: How many entries use each element
  • Sample entries: Examples of entries using the element
  • Unused elements: Elements safe to delete without affecting data
Example: Managing Grammatical Categories

If you want to split "Noun" into "Common Noun" and "Proper Noun":

  1. Open Tools → Ranges Editor
  2. Select the grammatical-info range
  3. Click Elements tab to see all grammatical categories
  4. Click Usage tab to see which categories are actually used
  5. Add new elements "Common Noun" and "Proper Noun"
  6. Update entries to use the new, more specific categories
  7. When ready, delete "Noun" and choose to migrate remaining references

Value Migration

When deleting a range element that's in use, you can:

  • Replace: Migrate all references to another element
  • Remove: Delete the value from all entries
  • Cancel: Keep the element if you change your mind

Workflow Features

Project Setup & Recommended Ranges Getting Started

When you first create a project, the Setup Wizard helps you configure basic settings such as the project name, source and target languages, and whether to install a minimal set of recommended ranges to bootstrap your project.

  • Project Wizard: Create a new project from the Settings → Projects page, or from the Settings page for the current project by using the Project Setup dialog.
  • Install Recommended Ranges: Choose this option to add a curated minimal set of ranges (grammatical-info, lexical relations, status, etc.) to quickly enable dropdowns and validation in the entry form.
  • Non-destructive install: The installer will not overwrite existing ranges. If ranges already exist, the installer refuses to run to prevent accidental data changes.
  • Post-install workflow: After ranges are installed, dropdowns and search filters will automatically populate with the new values; you may need to reload the UI.

Entry Ordering

Control how entries appear in your dictionary:

  • Order Attribute: Manual homograph numbering
  • Auto-ordering: Automatic alphabetical sorting

Metadata Tracking

Automatic and manual timestamp management:

  • Date Created: Automatically recorded when entry is added
  • Date Modified: Updated on every save
  • Date Deleted: Soft delete for workflow management

Multimedia Support

  • Illustrations: Add images to senses with captions
  • Audio Files: Link pronunciation recordings
  • Media Labels: Multilingual descriptions for media

FieldWorks Compatibility

This application provides 91% LIFT 0.13 compliance with full FieldWorks compatibility:

What Does 91% Compliance Mean?

The application implements 51 out of 56 LIFT 0.13 elements, covering all essential and professional features needed for lexicography. The 5 missing elements are low-priority features:

  • StText custom fields - Rich formatted text with complex styling (rarely used)
  • Entry-level media elements - Audio/video at entry level (use pronunciation media instead)
  • Variant phonetic forms - Specialized phonological variants
  • Complex span formatting - Advanced text styling in notes
  • Derivational affixes - Specialized morphological elements

Impact: These missing features don't affect typical dictionary workflows. All FieldWorks LIFT files import successfully with full data preservation for the 91% of elements implemented.

Import/Export Features

  • Lossless Import: Import LIFT files from FieldWorks/FLEx preserving 91% of elements
  • Round-trip Preservation: Export back to LIFT maintaining all implemented data
  • Range Files: Full support for LIFT-ranges custom classifications
  • Header Fields: Custom field definitions preserved
  • Tested Compatibility: Verified with real FieldWorks and LanguageForge files

Getting Started

Ready to start working with your dictionary data?

  1. Import your LIFT file: Go to Import/Export → Import LIFT
  2. Browse entries: Click on Entries to view your lexicon
  3. Edit an entry: Click on any entry to open the full editor
  4. Add new entries: Use the "New Entry" button
  5. Export: Save your work back to LIFT format anytime
